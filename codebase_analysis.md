### 2.4 Security Architecture

```
Request → CorsFilter → JwtAuthFilter → SecurityFilterChain → Controller
                          ↓
                   Extract Bearer token
                   Validate via JwtService
                   Load UserDetails from DB
                   Set SecurityContext
```

- **CSRF**: Disabled (stateless API)
- **Sessions**: STATELESS
- **Password encoding**: BCrypt
- **JWT Claims**: userId, email (subject), role
- **JWT Expiry**: 24 hours (86400000ms)
- **Role mapping**: `ROLE_CUSTOMER`, `ROLE_ADMIN` (via `hasRole('ADMIN')`)

### 3.3 Key Frontend Issues

> [!WARNING]
> **NO API Client exists**. There is no `apiClient.js`, no Axios, no fetch wrapper. Every page imports directly from `../../data/mockData.js`.

> [!WARNING]
> **AuthContext is completely fake**. The `login()` function creates a hardcoded user `"Nimal Perera"` with role `"customer"` — no actual API call, no JWT storage, no token handling.

> [!WARNING]
> **No loading states**. Pages render immediately with mock data — no `useEffect`, no `useState` for loading/error.

> [!WARNING]
> **Sidebar is customer-only**. No admin navigation links — admin routes exist but are not accessible via sidebar.

> [!WARNING]
> **Register flow is broken**. Just shows an `alert()` — no actual API call, no form validation.

> [!CAUTION]
> **No `fullName` or `confirmPassword` field on Register form**. Backend requires `fullName`, `confirmPassword` in `RegisterRequest`, but the frontend form only has `email` and `password`.
